# DBR進捗管理 v1.2 マニュアル

## 概要
`DBR進捗管理` は、DBR譜面のクリア状況、BP、スコア、メモをローカルで管理するためのWebアプリです。  
データはブラウザの `localStorage` に保存されます。

DBR難易度表や関連するIRを利用する際の記録整理を補助し、日々のプレー記録を必要に応じてDBR難易度表対応形式のJSONとして書き出せます。

## 画面構成
- ヘッダー部  
  テーマ切替ボタン、マニュアル、DBR IR、データ入出力ボタンがあります。
- 全体サマリー  
  難易度帯ごとの横棒グラフと、クリアランプ別の件数を表示します。三角トグルまたは `Overview` 文字で開閉できます。
- 曲一覧  
  現在の絞り込み条件に一致する曲カードを表示します。並び順と昇順・降順を切り替えられます。
- プレー記録  
  選択曲に対してクリア、BP、スコア、メモを入力します。
- 推移グラフ  
  BP推移とスコア推移を表示します。
- 選択曲の履歴  
  保存済み履歴を日付順で表示します。
- 絞り込み  
  フローティングの簡易絞り込みです。

## 初期準備
### 1. 難易度表を読み込む
- `難易度表読み込み` を押します。
- 曲一覧は難易度表データを基準に構築されます。
- 未読み込み時や前回の読み込みから12時間以上経過している場合は、ボタンが目立つ色で表示されます。

### 2. 既存データを取り込む
- CSV読み込み  
  縦データの履歴CSVを読み込みます。
- JSON読み込み  
  DBR難易度表対応形式のJSONを読み込みます。

## 曲一覧の見方
- 曲カードには以下が表示されます。
  - 難易度
  - SPLv.
  - 最高クリアランプ
  - Best BP
  - Latest BP
  - 最終プレー日
- 新規提案中の譜面は、曲カードが黄色背景になり、「新規提案中」バッジが表示されます。
- メモがある曲は、曲名の下にメモの先頭が小さく表示されます。
- カードを押すと、その曲がプレー記録に反映されます。
- 曲カードと選択中の曲カードの左側には、現在の最高クリアランプに応じた色が表示されます。

## プレー記録の付け方
### 1. 曲を選ぶ
- 曲一覧から曲カードを押します。

### 2. 入力する
- クリア
- BP
- スコア
- メモ

### 3. 保存する
- `保存する` を押します。
- 同一曲・同一日の記録がある場合は、その日の記録を更新します。
- 同じ日の履歴は、保存した順番が分かるように扱われます。

### 4. メモ
- メモは曲ごとに保存されます。
- `保存する` を押した時に保存されます。
- メモだけを入力して `保存する` を押すと、メモのみを保存できます。
- メモ欄を空にして `保存する` を押すと、メモを削除できます。

### 5. 当日の記録削除
- `当日の記録を削除` を押すと、選択曲の今日の記録だけ削除します。

### 6. 戻る
- `戻る` を押すと、選択中の曲カード位置へスクロールします。

## 提案フォーム
- 選択中の曲カードには、譜面の状態に応じて `新規提案`、`変更提案`、`おすすめ提案` のボタンが表示されます。
- 新規提案では、レベル、おすすめ度、コメントを送信できます。
- 変更提案では、変更後レベルと提案理由を送信できます。
- おすすめ提案では、変更後おすすめ度と提案理由を送信できます。
- 送信後は、送信先のシートを開いて内容を確認できます。
- すでに新規提案中の譜面では、提案内容を確認できる外部シートへのリンクが表示されます。

## 全体サマリー
### 開閉
- 見出し左の三角トグル、または `Overview` 文字を押すと、全体サマリーを開閉できます。
- 折りたたむと、枠の下側の余白は最小限に詰まります。

### 横棒グラフ
- 現在の絞り込み軸に応じて、難易度帯ごとの内訳を表示します。
  - Lv.
  - SPLv.
  - 片手Lv.
  - プレー日
- 棒の長さは一定で、内部をクリアランプ比率で積み上げています。
- プレー日で絞り込んでいる場合は、1本の棒が1日の記録を表します。
- プレー日がALLの場合は、履歴がある日のうち直近14日分を表示します。
- プレー日の範囲を指定している場合は、その範囲内で履歴がある日を表示します。
- プレー日で開始日だけを指定した場合は、開始日以降で履歴がある日を最大31日分表示します。
- プレー日で終了日だけを指定した場合は、終了日以前で履歴がある日を最大31日分表示します。

### クリアランプ枠
- `NO PLAY` などの枠を押すと、曲一覧側のクリアランプ絞り込みを切り替えます。
- 最後の1個を無効化しようとした場合は、全有効に戻ります。
- PCでは、ダブルクリックすると、そのランプだけ有効な状態にできます。
- スマホでは、枠を左にスワイプすると、そのランプだけ有効な状態にできます。
- 件数表示は、現在の絞り込み条件に追従します。
- `曲名` または `メモ` で絞り込んでいる間は、クリアランプ枠の操作は無効になります。

### フィルタ表示
- `フィルタを表示` から固定フィルタを開けます。
- 対応項目:
  - INFINITAS
  - AC収録
  - 未査定曲
  - おすすめ
  - リセット
- `リセット` は固定フィルタだけを初期化します。
  - INFINITAS
  - AC収録
  - 未査定曲
  - おすすめ
- 絞り込み軸、その値、クリアランプ状態は維持されます。

## フローティング絞り込み
### 絞り込み軸
- Lv.
- SPLv.
- 片手Lv.
- プレー日
- 曲名
- メモ

### 数値軸
- スライダーで単一値を選択します。
- スライダーを離した時に確定します。

### 曲名
- 曲名は完全一致、前方一致、部分一致の順で表示されます。
- 表記ゆれや記号違いもある程度吸収して検索します。
- フォーカスアウト時に検索が確定します。
- `解除` ボタンで直前の絞り込み状態へ戻せます。

### メモ
- メモの部分一致で検索します。
- フォーカスアウト時に検索が確定します。
- `解除` ボタンで直前の絞り込み状態へ戻せます。

### プレー日
- 開始日と終了日を指定して、プレー日の範囲で絞り込めます。
- 開始日だけ、終了日だけの指定もできます。
- プレー日を選んだ時は、前回選んだ範囲が復元されます。
- 前回の範囲がない場合は、履歴がある日の直近14日分を開始日とし、終了日は今日になります。
- 履歴がない場合は、開始日と終了日の両方に今日の日付が入ります。
- 表示中の範囲の横にある `戻す` で、既定の範囲に戻せます。
- 絞り込み軸ごとに、前回選んだ絞り込み内容が復元されます。

### キーボード操作
- `Q` でフローティング絞り込みを開閉できます。
- `S` で曲名検索へ移動し、検索欄をアクティブにできます。
- 入力欄、セレクト、ボタンを操作している間は、`Q` と `S` は動作しません。
- 入力欄やセレクトにフォーカスがある時は、`Esc` でフォーカスを外せます。
- `Esc` でフォーカスを外した場合、曲名・メモ検索やプレー日絞り込みは確定しません。
- 曲名またはメモ絞り込みを開いている時は、`Delete` で検索文字を消して検索欄をアクティブにできます。
- 検索欄にフォーカスがある時の `Delete` は、通常の文字削除として動作します。

## 並び順
- 曲名
- Lv.
- SPLv.
- 片手Lv.
- クリアランプ
- 最小BP
- 最新BP
- 最終プレー
- おすすめ
- メモ
- 曲一覧上部の `▲` / `▼` ボタンで昇順・降順を切り替えられます。
- 絞り込み軸ごとに、前回選んだ並び順が復元されます。
- 絞り込み軸を変更した時は、昇順で表示されます。
- 絞り込み軸をプレー日にした時の並び順は、初期状態では最終プレーになります。

## データ入出力
### CSV読み込み
- 縦データCSVを読み込みます。
- 同一 `title + date` の行は上書きされます。
- `lamp`、`bp`、`score` の一部が欠けていても取り込めます。
- `memo` 列があれば、曲メモも復元します。
- `timestamp` 列があれば、同じ日の履歴順も復元します。

### CSVを書き出す
- 現在の履歴を縦データCSVで書き出します。
- 出力列:
  - `timestamp`
  - `date`
  - `title`
  - `level`
  - `splv`
  - `lamp`
  - `bp`
  - `score`
  - `memo`
- `level` はDBR難易度表の難易度を出力します。
- `splv` はDBR難易度表のSPLv.を出力します。
- 記録のない曲でも、メモだけある場合は `memo` 専用行として出力されます。
- 書き出しファイル名の末尾には、書き出し日の年月日が `YYYYMMDD` 形式で付きます。

### JSON読み込み
- DBR難易度表対応形式のJSONを読み込みます。
- 読み込み時に記録日を指定できます。
- `lamp`、`bp`、`score` の一部が欠けていても取り込めます。
- 同一 `title + date` の既存記録は置き換えます。

### JSONを書き出す
- DBR難易度表対応形式のJSONを書き出します。
- `bp`、`lamp`、`score` は存在するものだけを書き出します。
- BP、スコア、クリアランプは、曲ごとの最高記録を出力します。
  - BPは最小値を出力します。
  - スコアは最大値を出力します。
  - クリアランプは最上位ランプを出力します。
- 書き出し時は、難易度表を再読み込みしてから書き出します。
- 難易度表の読み込みに失敗した場合は、現在保持している難易度表データで書き出します。
- JSON書き出しは、主にDBR難易度表対応形式・外部連携向けです。履歴全体のバックアップにはCSV書き出しを使用してください。
- 書き出しファイル名の末尾には、書き出し日の年月日が `YYYYMMDD` 形式で付きます。

### データ全削除
- 保存済みのプレー記録とメモを全削除します。
- 難易度表データは残ります。

## 注意点
- 曲一覧は難易度表データを基準にしています。
- 片手Lv.を絞り込み軸にした場合、片手難易度未設定曲は一覧と集計から除外されます。
- `曲名` または `メモ` を絞り込み軸にした場合、クリアランプ枠の操作は無効になります。
- `曲名` または `メモ` を絞り込み軸にした場合も、INFINITASとAC収録の固定フィルタは曲一覧に反映されます。
- テーマ設定はブラウザの `localStorage` に保存されます。
- このアプリは、各難易度表を置き換えるものではなく、それらを参照しながら個人のプレー記録を管理するための補助ツールです。
- データ保存先はブラウザごとの `localStorage` です。
- ブラウザや端末を変えると、保存データは自動では共有されません。

## 推奨運用
- 履歴管理の主データはCSVを使用する
- 外部連携や難易度表対応データ交換にはJSONを使用する
- 履歴全体のバックアップにはCSVを書き出して保管する
